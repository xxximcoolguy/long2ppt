CHAPTER 0205 / 18
FIRST DUEL · GUI 派双雄

Cursor vs Windsurf

PRODUCT ACursor · 增长之王

  • 4200 万月活 · 28% 付费率
  • VS Code fork + 优秀对话产品体验
  • 核心能力依赖底层模型（Claude / GPT）
  • 护城河在融化 · 体验是最易复制的维度

PRODUCT BWindsurf · 原生 IDE

  • 1800 万月活 · 24% 付费率 · L4 自主性
  • 从零写编辑器内核 · 1M token 上下文
  • Cascade 模式连续编辑数十文件 + 跑测试
  • 技术深度领先半年 · 但缺生态壁垒
COMPARE06 / 18
SECOND DUEL · CLI 派 vs 老牌

Claude Code vs GitHub Copilot

PRODUCT CClaude Code · 反 IDE 实验

  • 2400 万月活 · 32% 付费率（全行业最高）
  • 不做 IDE · 做 CLI + Skill + Hook + Agent
  • 在终端原生用户里渗透率极高
  • 哲学：编辑器是历史包袱，AI 是 shell 协作者

PRODUCT DGitHub Copilot · 失去主动权

  • 6500 万月活 · 但付费率仅 18% · L2 自主性
  • 2024 年起产品节奏明显落后
  • 无 Composer 对等 · 无 Agent 模式
  • 从领先者变追随者 · 微软体量保不出局

CHAPTER 0308 / 18
TREND 01 · 自主性等级

从光标补全到自主 Agent

任务长度的中位数从 2024 年的 90 秒延长到 2026 年的 27 分钟

Level 02
complete()

单行补全

机器在你写代码时建议下一个 token。代表：早期 Copilot。

Level 03
refactor()

多文件编辑

机器同时改动多个文件。代表：Cursor Composer / Zed AI。

Level 04
execute()

长任务自主

机器编辑+跑测试+提交，几乎无人介入。代表：Windsurf / Claude Code / Cline。

CHAPTER 0515 / 18
RISK MAP · 三大风险

风险按严重度排序

RISK 01 · CRITICAL

分发，不是产品

目标用户搜不到，做再好也是 zero。需要 Claude Code 生态 + 中文社群 + KOL 三层应对。

RISK 02 · HIGH

上游模型变化

所有 AI 产品都依赖底层模型，模型一变体验就抖。对策：model-agnostic 架构 + 多模型并行支持。

RISK 03 · MEDIUM

商业模式未经检验

22% 付费率虽高，没经过经济下行周期。2027 前必须把毛利率打到 60% 以上才安全。
