# Mode D · 主题自创流程

> 用户没给原文，只给主题方向（"做一个关于 X 的 PPT"）。**禁止跳过审核直接生成 HTML**。

## 触发条件

- 用户消息没有：本地文件 / 长文本 ≥ 300 字 / URL
- 用户只描述了一个主题或方向

## 与 frontend-slides 的边界判断

| 用户状态 | 走 | 原因 |
|---|---|---|
| 已决定要长文风格 PPT（带章节扉、双栏、金句墙）| 留本 skill 模式 D | 长文版式优势 |
| 完全没头绪，需要 mood-based 风格探索 | 推荐 `frontend-slides` | 它有 3 预览展示 |
| 不确定 | **主动问**："想要长文式分章节呈现，还是从零探索风格？" |

## Phase 1.8 流程（必经四问）

用 **一次** AskUserQuestion 把 4 个问题问完：

**Q1 · 主题与角度**（自由文本）
- 详细描述 PPT 主题、目标受众、核心观点

**Q2 · 内容范围**（multi-select）
- 背景 / 现状
- 核心观点 / 主张
- 关键数据 / 案例
- 对比 / 论证
- 方法 / SOP
- 金句 / 心法
- CTA / 结论

**Q3 · 大纲详略**：12 / 18 / 25 张

**Q4 · 知识来源**：
- Claude 自由生成（推荐）
- 我会补充关键资料（追加输入）
- 让我先看大纲再决定

## 生成草稿 → 审核 → 进入 Phase 2

收到回答后：

1. 按用户主题/角度，构思有内在逻辑的论述结构
2. 写满"假装是一篇 2000-5000 字长文"的草稿
3. 写入 `<output_dir>/source.md`，`meta.json` 标 `"format": "claude-authored"`
4. ⚠️ **必须先把草稿摘要展示给用户审核** —— 列出 5-10 个章节要点 + 核心论点
5. 用户确认/调整后才进入 Phase 2（标题候选 + 大纲）

## 防呆铁律

- ❌ **不要凭主题直接出 PPT** —— 没有草稿审核 = 凭空跑偏
- ❌ **不要省略 Phase 2 标题候选** —— 即使 Claude 自创，也要让用户选标题
- ✅ 草稿审核失败时（用户说"跑偏了"），**回到 Q1 重新写**，不要硬改
