# Dieter Rams · 视觉操作系统

> 「**Weniger, aber besser.**（少，但更好）」 — Dieter Rams

| 项 | 内容 |
|---|---|
| **人物** | Dieter Rams (b. 1932) |
| **代表作** | Braun T3 / SK4 / T1000 / ET66 · Vitsoe 606 通用书架 · *Less and More* (2014) |
| **影响** | Jony Ive 师承（iPod 致敬 T3 / iOS 计算器致敬 ET66） |

---

## 1. 视觉哲学

### 核心信条：Weniger, aber besser
不是"少"，是"少到刚好够"。

### 十条好设计原则（一手 · Vitsoe 官方）
1. 创新（Innovative）
2. 有用（Useful）
3. 美（Aesthetic）
4. 让产品可被理解（Understandable）
5. **不张扬（Unobtrusive）** —— 对版式最关键
6. 诚实（Honest）—— 不假装是它不是的东西
7. 经久（Long-lasting）—— **反 trendy**
8. 每个细节都贯彻（Thorough）
9. 环境友好（Environmentally friendly）
10. **尽可能少的设计（As little design as possible）** —— 最著名

### 翻译到平面/版式的三句话哲学
- **页面是仆人不是主角**：版式服务阅读，不喧宾夺主（Unobtrusive）
- **每一个像素都要有功能理由**：装饰必须能解释存在的必要性
- **能用 3 年不过时才合格**：拒绝当下 trendy 配色 / 装饰潮流

### 与 Apple/Jony Ive 的传承
Braun T3 收音机（1958）→ iPod（2001）共享圆形控件、矩形比例。Braun ET66 计算器（1987）→ iOS 计算器 App 按钮布局几乎一对一致敬。

## 2. 版式操作系统（七条硬规则）

### H-1 · 网格即骨架（来自 Vitsoe 606 E-Track 思想）
- 12 列或 8 列基础网格，所有元素必须吸附
- 模块化：标题块 / 正文块 / 图块都是可拆卸的"shelf"
- 一页只用 2-3 个网格单元宽度，**不挤满**

### H-2 · 留白是结构，不是空地
- 页面边距 **≥ 8% 屏宽**
- 行距 1.5-1.7 倍字号
- 元素间距用 **8/16/24/32/64 等倍数节奏**

### H-3 · 字号系统：严格 3-4 级
- H1（封面/章节）/ H2（页标题）/ Body（正文）/ Caption（注释）
- 比例 **4 : 2 : 1 : 0.75**（如 64 / 32 / 16 / 12）
- **禁止中间级**（H1.5 / Body+）

### H-4 · 一页一事（Thorough down to the last detail）
- 每页只承载一个核心信息
- 标题 + 单一论点 + **至多一组**配图/数据
- 反对"信息塞满 = 价值高"

### H-5 · 直角为主，圆角为功能
- Rams 偏直角矩形（T1000 收音机）
- 工程细节用小圆角（T3 / ET66 按钮）
- 翻译：卡片用 **0px 或 2-4px** 微圆角，**禁止 16px+ 网红圆角**

### H-6 · 不张扬的层级（Unobtrusive）
- 用**字重 + 字号 + 间距**做层级
- **不用**装饰线 / 阴影 / 渐变 / emoji
- 强调用"留白包裹 + 颜色"而非"加粗加大加边框"

### H-7 · 一切对齐
- 左对齐为主（中文阅读节奏）
- 居中只用于封面 / 大标题
- 所有元素必须能解释自己为什么在这个位置

## 3. 颜色与字体启发式

### 字体（一手历史）
Braun 在 SK5 之后系统使用 **Akzidenz-Grotesk**（1898 年德国 Berthold 字体厂，比 Helvetica 早 60 年）。Rams 个人主导的产品几乎只用 Akzidenz-Grotesk，且多为**全小写**。Helvetica（1957）是 Akzidenz 的精炼版，Braun 也部分使用。

**PPT 落地字体推荐：**
- 西文优先：**Inter / Söhne / Neue Haas Grotesk Display / Helvetica Now**（最接近 Akzidenz 精神）
- 西文次选：Helvetica Neue / Arial
- **中文：** 思源黑体 / 苹方 / Inter + 思源黑体组合（克制、几何、无衬线）
- **禁止：** 圆体（Hiragino Maru、得意黑）/ 衬线体作正文 / 装饰字体

### 颜色系统（一手 · Braun 产品色卡）

| 角色 | 色值 | 用途 |
|------|------|------|
| Braun White | `#F0EDE5`（暖白，**非纯白**） | 主背景 |
| Braun Grey | `#8A8A87` | 次要面 / 分隔 |
| Warm Black | `#1A1A18` | 文字（**非纯黑**） |
| Signal Red | `#C02820` | 仅电源/警示按钮 |
| Function Yellow | `#D4B018` | 仅功能性强调 |

### 配色铁律
- **黑白灰打底 + 一个低饱和强调色**，强调色面积 **≤ 5%**
- 强调色必须有功能（标记重点、CTA、警示），不能纯装饰
- **禁止 trendy 渐变**（紫粉、青绿、莫兰迪全配色板）
- **禁止纯黑 #000 + 纯白 #FFF** —— Rams 偏暖灰白和暖黑

## 4. 视觉禁忌（强禁忌，违反即败）

1. ❌ **装饰性元素** —— 没有功能的线、点、几何图案、emoji、表情符号
2. ❌ **trendy 配色** —— 渐变紫粉、霓虹、网红莫兰迪、当年流行色
3. ❌ **视觉张扬** —— 大字号填满屏、对比度爆炸、动画炫技
4. ❌ **过度圆角** —— 16px+ 的卡片圆角（属于 2020s iOS trendy，非 Rams）
5. ❌ **drop shadow + neumorphism** —— 不诚实（Honest 原则反对"假装立体"）
6. ❌ **多字体混搭** —— 一个西文 + 一个中文，最多两个家族
7. ❌ **居中流水账版式** —— 全页居中是 PPT 模板病，Rams 强左对齐
8. ❌ **"为了不同而不同"** —— 每页用不同版式 = 装饰性变化，违反 Thorough
9. ❌ **填满** —— 留白率 < 30% 直接违规
10. ❌ **情绪化色彩选择** —— 不靠颜色"煽情"，靠内容本身

## 5. 局限性

### 不适合场景
1. **情绪驱动内容** —— 婚礼请柬、悼念文、抒情诗。中性色 + 几何字体压不住情绪密度
2. **文化娱乐产品** —— 电影海报、音乐 MV、游戏宣发。需要张扬视觉钩子，Unobtrusive 是反效果
3. **大众营销 / 社媒爆款** —— 小红书封面、抖音封面、电商 banner。Rams 克制会被算法淹没
4. **儿童 / 教育低龄** —— 需要饱和色 + 圆润形状 + 趣味装饰
5. **奢侈品 / 工艺品** —— 需要质感叙事（金箔、肌理、衬线），Rams 的工业冷静感弱化奢华
6. **中文密度问题** —— 中文字符天然比拉丁文字密度高 30-50%。**中文落地需调整：** 行距加大到 1.7-1.9，字号比西文小一档，**单页字数严格控制 ≤ 80 字**
7. **与 Carson / Scher 等 emotional design 的根本冲突** —— Rams 信"设计是仆人"，Carson 信"设计是表达"，两者不可调和

### 批评共识
当代批评认为 Rams 的"timeless"在精英化、缺乏温度、对多元文化适应性弱；同时被广泛误读为"剥掉一切 = Rams"，丢失了 functionality preservation 的本意。

## PPT 落地强制规则

| 维度 | 硬规则 |
|------|--------|
| 字号 | 严格 4 级（如 64/32/16/12），不准插入中间级 |
| 颜色 | 黑 + 白 + 灰 + 1 个低饱和强调色，强调色面积 ≤ 5% |
| 留白 | 页面边距 ≥ 8% 屏宽，整页留白率 ≥ 35% |
| 网格 | 12 列网格，元素必须吸附，每页用 ≤ 3 个网格单元宽 |
| 装饰 | 0 装饰元素（无线、无图标、无 emoji） |
| 圆角 | 0-4px，**禁止 16px+** |
| 字体 | 西文 Inter/Helvetica，中文思源黑/苹方，**最多 2 个家族** |
| 阴影 | 0 drop shadow，0 neumorphism |
| 每页信息 | 一页一事，**单页中文 ≤ 80 字** |
| 对齐 | 强左对齐，居中仅限封面 |
| 节奏 | 间距用 8/16/24/32/64 倍数 |
